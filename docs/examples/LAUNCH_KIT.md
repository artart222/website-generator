# Launch Kit

Use this file as the copy bank for your GitHub repo, profile, and launch posts.

## Positioning

Core angle:

`website-generator` is a Python static site generator for developers who want the simplicity of Markdown with the flexibility of themes, collections, plugins, and optional hybrid frontend tooling.

Short pitch:

It is not trying to be the biggest static site framework. Its appeal is that the architecture is understandable, hackable, and easy to extend if you want a generator you can actually own.

## GitHub Repo Description

Pick one:

1. Developer-first static site generator in Python with themes, collections, plugins, and optional React export.
2. A Python static site generator with Markdown content, theme packages, project overrides, and hybrid frontend support.
3. Build blogs, docs, and portfolio sites from Markdown using a Python generator with themes, plugins, and collections.

## README Above-The-Fold Goals

Your README should answer these quickly:

1. What is it?
2. Why is it interesting?
3. Why would someone pick it over a generic toy project?
4. How fast can they try it?
5. What proof is there that it works?

The current README now covers those, so the next improvement after launch is usually a short demo GIF near the top.

## Social Preview Asset

Recommended GitHub social preview image:

1. Start with `Demo-Screenshot.png`
2. Crop it to a wide banner, ideally `1280 x 640`
3. Add a dark overlay so text stays readable
4. Put this headline on top:
   `Build static sites with themes, plugins, and collections`
5. Put this subhead under it:
   `A developer-first static site generator in Python`

## Launch Checklist

Before posting:

1. Add the repo description and topics on GitHub
2. Upload a social preview image
3. Pin the repo on your GitHub profile
4. Make sure Actions are green
5. Use one clear screenshot or GIF in the README and posts
6. Frame the value clearly:
   not just "I made a generator", but "I made a generator that is readable, extensible, and good for developers who want control"

## Feedback Prompt

If you want a stronger response rate, end posts with a specific ask:

1. `Which feature would make this more useful to you?`
2. `Would you rather see more themes, more plugins, or a cleaner project scaffold next?`
3. `If you've built your own tools before, what would you improve in this architecture?`
